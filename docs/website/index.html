A VFX-grade image sequence & video viewer. Built for VFX review, not generic playback.

Flick Player reads what your pipeline produces — multichannel EXRs, OCIO-aware floats, sparse sequences with holes, mp4 / mov / ProRes clips with audio, multiple takes side by side — and gives you the tools you need to actually review them. Stack, trim, annotate, comment, export, share.

Real-time playback

Async RAM cache + OpenGL viewport. Smooth scrubbing on 4K EXRs at 24 fps. Worker pool auto-tunes to your CPU + GPU.

Stack & mount

Drop multiple sequences to compose a layered review. Reorder by drag, trim IN/OUT on the timeline, slide each layer's offset. Image sequences and video clips mix together.

A/B compare

Two layers, four modes: vertical wipe, horizontal wipe, opacity blend, solo-B. Drag the seam live — pure shader, no re-decode. Bake the wipe into your export too.

Smart cache

Multi-version key keeps a snapshot per channel selection — switching AOVs back and forth is an instant hit, not a re-decode. Smart reload diffs file mtimes; force reload (Ctrl+Shift+R) drops everything.

Background channel pre-cache

Once the active channel is cached, idle workers quietly pre-decode every other group (RGB, albedo, normal, Z…). Per-row progress bar in the channel menu. Suspended during playback so the GIL stays clean.

OCIO color

ACES 2.0 CG built-in. Source / display / view round-trip on the GPU shader — toggling is free. Auto-detect from EXR metadata (Nuke, Arnold, V-Ray, Houdini), with a per-pipeline override for unmarked sources.

Annotations

Pen / eraser / 7-color palette / per-frame undo. Persistent strokes — and a Google-Meet-style ghost mode that auto-fades.

Comments

Per-frame text threads beside the visual annotations. Author auto-filled from OS user, blue dot on the timeline. Round-trips through the same sidecar JSON.

Video & audio

Drop mp4 / mov / mkv / ProRes / DNxHR clips alongside your EXR layers — PyAV-decoded video with synced audio output. Per-layer mute / solo and wall-clock-driven playback so A/V never drifts.

Multichannel EXR

Channel selector for AOVs, depth, normals, cryptomattes — per layer. R/G/B mute + solo-alpha view (Nuke / RV convention). Pickable transparency background: checker, black, mid-grey, white.

Export

Full-sequence to PNG / JPG / EXR / TIFF or H.264 / H.265 / ProRes / DNxHR / VP9 / FFV1 / v210. Bake annotations and the live A/B compare wipe. Custom output filename. Save Frame As… for one-shot snapshots.

Sessions

Save the full layer stack — sources, offsets, trims, channels, alpha modes, audio gain — to a single .session JSON. Drop the file back on the viewer to restore everything.

Hardware-adaptive RAM

Auto-tunes cache budget + worker count from your GPU / total RAM. Status bar shows live cache used / budget · system free so you can spot pressure without opening Task Manager. Re-tunes on every project open.

Two takes, one wipe.

Pick layer A, pick layer B, drop the seam where you want it. The compare overlay lives entirely in the GPU shader — dragging the seam is a uniform update, not a re-decode. Four modes cover every comparison gesture you'd otherwise hop between two viewers for.

  • Vertical wipe — left half from A, right half from B. The classic plate-vs-comp gesture.
  • Horizontal wipe — top from A, bottom from B. Useful when the difference is along a horizon.
  • Opacity blend — linear mix at the seam value. 0 = pure A, 1 = pure B.
  • Solo B (A↔B button) — temporary "show full B" override without leaving the chosen blend.
  • Right-click drag in the viewport moves the seam live; W toggles compare on/off; Ctrl+W swaps A and B.
  • Bake into your export — tick "Bake A/B compare overlay" in the export dialog and the live wipe is reproduced for every frame written to disk.

A timeline that tells you what's going on.

One bar carries every dimension of the sequence's state — at a glance, you know which frames are cached, which failed to decode, which fall in a multi-layer gap, where you've left a note, what's in/out range, and where the playhead is.

  • Translucent orange runs = frames in RAM. The bar shows your prefetch high-water mark.
  • Red runs = source file missing or unreadable. They override the cache run on the same slot so a hole is impossible to miss.
  • Mid-grey runs = no layer covers — Flick plays through and the viewport paints a checkerboard with the missing filename baked in.
  • Green triangles = annotated frames. Blue dots = frames with comments.
  • Ctrl + click sets and drags the in / out points — like Nuke.
  • The cursor scrubs outside the in/out range; playback snaps back inside on press.
  • Frame ↔ timecode labels swap with Ctrl+T — same bar, both reading conventions.
  • Minimal mode in fullscreen strips everything but the rail and the playhead, so playback stays the only thing on screen.

Layers, trim and offset.

Drop image sequences, video files (mp4 / mov / mkv / m4v / avi) or single stills and Flick Player stacks them. Each layer carries its own offset on the master timeline, its own IN / OUT trim, its own channel selection, its own alpha-composite mode and its own audio state. Drag the body to slide, drag a handle to trim — the right edge stays put on IN-trims, just like a real NLE.

  • Drop on the viewer = replace the focused sequence. Drop on the panel = add as a new layer above.
  • Drag the row vertically to reorder. A drop indicator shows where it'll land.
  • Trim handles snap to the playhead, master IN/OUT and other layers' edges.
  • Ghost overlay (dashed orange) shows source frames hidden by the trim — drag the handle back outwards to reveal them.
  • Per-layer T / αS toggles control alpha-over compositing and straight-vs-premultiplied alpha handling.
  • M / S audio buttons on each video layer — mute or solo, like a mixer. PlayerController stays in wall-clock mode so A/V never drifts.
  • Still layers get an adjustable hold duration so a single PNG / JPG can stretch over as many master frames as the review needs.
  • 👁 visibility per layer — temporarily hide a pass without losing its slot; the cache keeps its frames around for when you toggle it back on.

The cache that remembers your channels.

Most players treat their RAM cache as a flat master_frame → pixels map. Switch a channel and the whole range invalidates. Flick keys on (master_frame × signature) instead, where the signature captures the chain × channel selection. Switch RGB → albedo → RGB and the second RGB is an instant cache hit, not a 1000-frame re-decode.

  • Background channel pre-cache — once the active channel is fully cached, idle workers quietly pre-decode the focused layer's other groups (RGB beauty, AOVs, normals, Z…) into their own cache slots. Switching becomes free.
  • Per-row progress bars in the channel menu show each group's fill in real time. Same translucent-orange idiom as the timeline cache bar — visible right on the dropdown button while it's closed.
  • Keyboard navigation in the channel menu — ↑ / ↓ walk through groups without leaving the keyboard. Pairs naturally with the cached-snapshot model: every step is instant.
  • In/out range first — alt prefetch front-loads frames inside your playback range, then fills the rest of the nav range.
  • Suspended during playback so the GIL stays clean — workers can't be preempted mid-decode, so we hold off until you pause. No more 1-2 frame stalls when an alt steals time during a scrub.
  • Smart eviction — when RAM gets tight, stale-signature snapshots and hidden-layer contributions go first, then live frames by distance from the playhead. The view you're looking at is always the last to leave.
  • Smart reload (Ctrl+R) re-decodes only files whose mtime changed. Force reload (Ctrl+Shift+R) drops everything for the rare case where files were overwritten without an mtime bump.

Two ways to annotate.

Persistent = strokes saved to a sidecar JSON, frame-keyed, ready to share. Ghost mode (👻) = strokes fade out on their own, perfect for live screen-share commentary in Meet / Discord. The toolbar's outer border turns cyan when ephemeral mode is active so the state is unmistakable.

  • 7-color review palette tuned for VFX (red / yellow / green / blue / orange / b&w).
  • Per-frame undo — Ctrl+Z only undoes the current frame's edits, no surprise jumps.
  • Eraser deletes whole strokes by hit-testing the polyline, not pixel-level.
  • Three fade presets in ghost mode: court ~2 s, moyen ~5 s (default), long ~10 s.
  • 👁 Show during playback — toggle visibility in real time without losing your work.
  • Bake into export if you want a self-contained MP4 to send a colleague.

Real OCIO color, on the GPU.

Flick Player ships with the ACES 2.0 CG built-in config. Point it at your studio's custom config.ocio via the $OCIO env var if you have one. The display transform runs on the fragment shader, so toggling source / display / view is free — no cache invalidation, no re-decode.

  • Auto-detect input colorspace from EXR header attributes — the loader sweeps for colorspace / color_space keys written by Nuke, Arnold, V-Ray, Houdini, and matches them against the active config's role and alias table.
  • Unmarked-EXR override: when no metadata is found, the Color panel lets you pick the default colorspace per pipeline (ACEScg, scene_linear, Linear Rec.709…) and remembers it across sessions — no more tagging every shot by hand.
  • Per-image exposure nudge with + / - (±0.25 stops), plus gamma trim — both live in the shader.
  • Per-layer channel selection — pick a single AOV / depth / normals layer and view it as RGB; pre-cached in the background per layer with its own master-frame entry.
  • RGBA mute: toggle channels off in the shader without invalidating the cache. Solo-alpha shows the alpha channel as grayscale (white = opaque, black = transparent), Nuke / RV convention.
  • Transparency BG picker next to the RGBA buttons — checker / black / mid grey / white. The fragment shader branches on a uniform; switching is free and the choice persists across sessions.

Per-frame comments for the team.

Annotations are visual; comments are textual. A side panel beside the viewer holds a thread of notes attached to whatever frame the playhead is on — perfect for "the lighting is too cool" feedback that doesn't need a stroke.

  • Author auto-filled from the OS username — no login flow.
  • Created / edited timestamps tracked per comment.
  • Blue dots on the timeline mark every commented frame for one-click jumps.
  • Same JSON sidecar as annotations — one file travels with the sequence.
  • The Comments tab shares the side dock with Color; flip with one click.

Export to anything.

Send your sequence as an image-per-frame run for the next tool in the pipe, or bake it into one of eight video codecs to ship a review reel. The live A/B compare wipe bakes through too. Every option persists between sessions.

  • Custom filename: empty falls back to the source basename, otherwise drives the output (image seq → {stem}.NNNN.ext, video → {stem}.ext). Versioned bundle dirs by default.
  • Range: full sequence, in/out, or custom from / to. Resolution: source / 720p / 1080p / 4K UHD / custom (with aspect lock). Frame rate: source / 23.976 / 24 / 25 / 29.97 / 30 / 60 / custom.
  • Color: bake the OCIO display transform, or keep the linear buffer untouched (EXR / TIFF passthrough).
  • Annotations: bake into pixels + optional sidecar JSON copy alongside the export.
  • Bake compare overlay: when the live wipe is on, the export reproduces it for every frame. Untick to ship the active sequence's clean composite instead.
  • Codecs: PNG / JPG / EXR / TIFF for image sequences; H.264 / H.265 / ProRes 422 HQ / ProRes 4444 / DNxHR / VP9 / FFV1 / v210 for video.
  • Missing-frame policy (see below): never let a hole abort an overnight render again.

Save Frame As… for one-shot stills.

Need a single image to drop in a review email or a Slack thread? File → Save Frame As… snapshots the viewer with the same compare and annotation toggles as the full export — no need to spin up a 1-frame range. Picks up the active OCIO display transform too, so what you see is what gets written.

  • Formats: PNG / JPG / TIFF / BMP / WebP. Defaults to PNG with the source basename + master frame number.
  • Same toggles as full export: bake annotations, bake the A/B compare wipe, bake OCIO display.
  • Persistent path — the dialog remembers the last folder you wrote to, per session.

Three policies for missing frames.

Found a hole in the sequence at 03:12 AM during the overnight render? Pick the policy on the way out so the export decides what to do without you.

  • Abort — fail loud at the first missing frame. Default; safest for finals.
  • Black — substitute a black frame. Quick when the gap is near a cut and you're shipping a draft.
  • Placeholder — write the rich "MISSING FRAME" graphic. Best for a dailies pass — reviewers can't miss the issue.

What's new.

Every public release, what changed, why it changed. Links go to the matching tag on GitHub.

v1.4.0 (unreleased) 2026-05-08 Preferences dialog, OCIO hot-reload, splash refactor.

New

  • Preferences dialog — File → Open Preferences… (Ctrl+,) opens a sidebar+stack settings window. First section: Color Management, with three modes for resolving the OCIO config at boot — Default (built-in ACES), From $OCIO, or Custom config file… (browse to any .ocio on disk). The dialog's structure is set up to accept further sections (Playback, Cache, Annotation…) by adding one widget.
  • OCIO config hot-reload — applying a new config in Preferences swaps the running OCIOManager in place: ColorPanel combos repopulate, the GPU shader rebuilds against the new transform, persisted source / display / view picks that no longer exist in the new config are dropped silently. Restart no longer required. Banner turns green with the loaded config name and a summary of which picks were preserved vs reset.
  • Cream-on-orange channel menu typography — when the cache-fill bar is half-filled behind a channel row, the label text repaints in #FFE5C0 (the same warm cream the bottom info band uses) clipped to the filled region, so the row stays legible all the way across the bar. Outside the bar the standard theme colour shows through, so typography is untouched until loading begins.
  • QSplashScreen at boot — the previous PyInstaller bootloader splash used Tk, which isn't DPI-aware on Windows and visibly shrunk + repositioned the splash mid-boot on HiDPI displays. The new splash is a Qt QSplashScreen reading a checked-in PNG asset; DPI-aware, no shrink. Trade-off: ~1-2 s appearance latency (PySide6 import) instead of the bootloader's instant paint.

Fixes

  • Cache-fill bar on channel rows starts a few pixels before the label's first character (in the radio↔label gutter) so the orange wash never crashes flush against the leftmost glyph.
  • Build chain — bundle build now produces a working EXE on miniforge Windows: tkinter excluded again (saves ~5 MB), Tcl/Tk PATH workaround dropped, OIIO / OpenEXR / Imath DLLs explicitly pulled from $CONDA_PREFIX\Library\bin\ so the bundle is no longer missing them at startup.
v1.3.0 2026-05-08 A/B compare, smart cache, color profiles.

New

  • A/B compare overlay — pick two layers and split the viewer with a vertical wipe, horizontal wipe, opacity blend, or solo-B mode. The seam is draggable (right-click + drag), keyboard-nudgeable, and persists across layer picks. Compositing runs on the fragment shader so seam updates are uniform-only — no numpy compose, no cache invalidation.
  • Compare band in the menu bar — A / B picker pills, mode pills (Vert / Horiz / Opacity / Solo-B), and a swap button live next to the File menu. The band shrinks with the window width and never overlaps menu hit-targets.
  • Smart cache with multi-version key — the cache key is now (master_frame, signature) where the signature folds in layer id, label, active channel, premult flag, offset and in-frame. Switching channel or stack order keeps the previous version's entries hot for instant flip-back; eviction prefers older versions of off-screen frames first.
  • Background channel pre-cache — alt-channel reads queued at low priority during idle on every visible layer (top-down stack order), so swapping the active channel on any layer paints in cached frames immediately. Suspended automatically during playback to free the GIL, resumed on pause.
  • Per-row cache progress — the channel menu shows a dotted progress bar per channel and the layer button mirrors the active channel's coverage. The status bar tracks total RAM headroom.
  • Color metadata sweep — the loader matches EXR header attributes against Nuke / Arnold / V-Ray / Houdini patterns (substring sweep on colorspace, color_space, role / alias names) before falling back to filename heuristics.
  • Unmarked-EXR override — pick the default colorspace per pipeline in the Color panel (ACEScg, scene_linear, Linear Rec.709…) for EXRs that ship without metadata; persisted across sessions.
  • Solo-alpha + transparency BG picker — the alpha-only mute swaps in a checkerboard or a solid background colour so the matte reads correctly against the viewport.
  • Force reload — Ctrl+Shift+R clears the cache for the active master frame and re-reads from disk, even for frames already resident. Useful when artists overwrite shots in place.
  • Save Frame As… — File → Save Frame As… snapshots the viewer to PNG / JPG / TIFF / BMP / WebP with the same compare and annotation toggles as the full export.
  • Custom export filename + bake compare overlay — empty falls back to source basename, otherwise drives the output ({stem}.NNNN.ext for sequences, {stem}.ext for video). The live A/B wipe bakes through every frame; untick to ship the active sequence's clean composite.
  • Single-image layers — drop a still on the viewer and it becomes a layer with adjustable hold duration. Great for slates, refs and frame holds in a comp pass.
  • Versioned bundle directory — dist/FlickPlayer_v<version>/ instead of dist/img_player/, so multiple builds coexist.
  • No console window — the .exe ships without a stray cmd window. Logs are written to %LOCALAPPDATA%\img_player\flick.log with rotation.
  • Splash screen at startup — PyInstaller bootloader splash with three status milestones (Configuring logger…, Detecting hardware…, Loading Flick Player…) so cold launch on slow disks doesn't look frozen.
  • Taskbar icon — Windows AppUserModelID set explicitly so the taskbar shows flick.ico instead of the generic Python feather.
  • Channel arrow navigation — Up/Down arrows in the channel menu walk through the channel list without leaving the keyboard. Annotation close button + fullscreen polish ship in the same pass.
  • Missing-frame placeholder bakes the filename — the rich error graphic now writes the missing frame's expected filename into the overlay, so reviewers can spot which frame to re-render.

Fixes

  • Compare seam preserved across layer-picker changes; auto-disabled on File → New so a fresh session never starts in compare mode.
  • Layer-picker visible checkbox now toggles correctly on click instead of swallowing the event.
  • Export keeps partial files on cancel and surfaces colour-coded outcomes (success / cancelled / error) instead of a silent dialog dismiss.
  • Cache handles 1-channel mask buffers and 2-channel L+A buffers without raising; 5-channel crash on AOV groups with a bare alpha is gone.
  • Boot tolerates the missing QScreen.colorSpace attribute on PySide6 6.11 Windows builds.
  • Cleanup pass — ruff fixes, helper de-duplication, stale comments hunted out across the cache and compare modules.
v1.2.0 2026-05-05 Bottom info band.

New

  • Orange info band at the bottom of the viewer (PDPlayer-style HUD). Four readouts in one strip: image dimensions, session fps, source-frame number on the topmost visible layer (matches the file name on disk), and absolute master-timeline frame. Toggleable via View → Show info band, the ⓘ pill in the timeline gutter, or Ctrl+I; on by default.
  • Info band readouts follow the timecode toggle (Ctrl+T) — Layer / Frame swap to HH:MM:SS:FF in step with the timeline ticks and the transport's frame display, so the three readouts can never disagree on units.
  • launch_app.bat at the repo root — double-click to activate the conda env and start the GUI without opening a Miniforge Prompt by hand.

Fixes

  • OpenImageIO import is robust against PATH pollution from other VFX tools (mrViewer, RV, Nuke runtime…). Setting OIIO_LOAD_DLLS_FROM_PATH=0 in img_player/io/reader.py stops OIIO from registering every PATH entry as a DLL search dir, which used to make it pick up wrong-version libheif / OpenEXR / libavif DLLs and fail with "DLL load failed: La procédure spécifiée est introuvable".
  • Info band readouts follow timeline scrub in real time — pushed straight from _on_scrub_requested like the transport's frame display, bypassing the 20 ms scrub-debounce so the digits track the cursor under the mouse instead of trailing it.
v1.1.0 2026-05-05 Video & audio playback.

New

  • Video playback — drop .mp4 / .mov / .mkv / .m4v / .avi files alongside your image sequences. PyAV-backed decoder, threaded with a 2-frame-ahead prefetch so play and scrub stay smooth on H.264 / H.265 / ProRes / DNxHR.
  • Synced audio — sounddevice OutputStream open from boot to shutdown (zero play-time latency). Per-layer Mute / Solo toggles on every LayerRow, exclusive solo (DAW idiom), per-layer gain. Speed ≠ 1.0× mutes (no pitch shift).
  • Wall-clock-driven tick — the PlayerController now anchors playback on time.monotonic instead of "+1 per QTimer firing". Audio (which plays at PortAudio wall rate by construction) and video stay aligned long-term, even when the GUI thread hiccups.
  • Multi-source drag-and-drop — drop a folder of EXRs and several .mp4 in the same gesture, the picker shows a hierarchical tree per folder; tick exactly the sequences you want as layers.
  • Drop a .session on the viewer — full-window OPEN SESSION overlay (purple) instead of the per-zone REPLACE / ADD overlays, so the gesture's intent is unambiguous.
  • Per-layer audio fields in .session files — audio_solo, audio_mute, audio_gain persist with the rest of the layer state.
  • Inno Setup installer template in installer/flick.iss — wrap the PyInstaller bundle into a per-user Windows installer with Start menu shortcut, .session file association and a clean uninstaller.

Fixes

  • Channel-menu radio & checkbox indicators now visible against the dark theme — the global QWidget {{ border: none }} rule was hiding them. Multi-tile contact sheet was unreachable as a result.
  • Multi-tile contact sheet works again — single-layer selections with a tile-set now bypass the multi-layer composite path that was rejecting buffers wider than RGBA (Unsupported channel count: 7).
  • Fresh layer load no longer auto-checks tile groups from the previous session's prefs — tile sets are session-bound now (the active group + layout mode + label visibility are still preference-restored).
  • Scrub during play auto-pauses for the duration of the drag — used to thrash the threaded video decoder with backward seeks every tick.
  • Audio re-seeks correctly when the active layer's offset changes; coverage transitions during play (entering / exiting a clip) silence the feeder so the buffer doesn't drift ahead of the playhead.
  • Audio pitched down one octave on stereo decode — PyAV resampler was configured for interleaved flt when the consumer expected planar; switched to fltp.

v1.0.0 2026-04-29 First public release. Multi-layer.

New

  • Multi-layer stack — drop multiple sequences on top of each other, reorder, toggle visibility per layer. Topmost-visible wins for each master frame; αS turns on alpha compositing per layer.
  • Layer panel under the timeline — one row per layer with eye / T / αS toggles, drag to reorder, drag the bar to nudge offset, drag the handles to trim IN / OUT.
  • Per-layer channel selection — each layer remembers its own RGB / RGBA / Y / N.X choice; switching focus repopulates the channel menu without invalidating other layers' caches.
  • Sessions — File → Save / Open session writes the whole stack (paths, offsets, trims, channels, αS, visibility) to a .session JSON. Reopen later, get the same review setup back.
  • Drop modes — drag-and-drop now prompts: replace the focused layer, add as a new top layer, or open in place.
  • Per-row delete on every LayerRow; full undo/redo coverage on layer add / remove / reorder / drag.

  • Viewport stays black on first sequence load until you scrub — fixed; the first frame now renders as soon as the cache lands.
  • OUT trim was getting reset to the drag-start position by the snap formula — fixed.
