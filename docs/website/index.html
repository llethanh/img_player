Built for VFX review, not generic playback.

Flick reads what your pipeline produces — multichannel EXRs, OCIO-aware floats, sparse sequences with holes — and gives you the tools you need to actually review them: annotate, comment, export, share.

Real-time playback

Async RAM cache + OpenGL viewport. Smooth scrubbing on 4K EXRs at 24 fps.

OCIO color

ACES 1.3 / 2.0 built-in configs, full source / display / view round-trip on the GPU shader.

Annotations

Pen / eraser / 7-color palette / per-frame undo. Persistent strokes, plus a Google-Meet-style ghost mode.

Multichannel EXR

Channel selector for AOVs, depth, normals, cryptomattes. Per-channel R/G/B/A mute toggles.

Smart cache

Reload only re-decodes files whose mtime changed. Missing frames serve a checkerboard placeholder so playback never freezes.

Export

PNG / JPG / EXR / TIFF sequences or H.264 / H.265 / ProRes / DNxHR / VP9 / FFV1 / v210 video, with annotations baked in.

Comments

Per-frame text threads alongside the visual annotations. Saved next to the sidecar.

Sidecar JSON

Annotations + comments live in one human-readable JSON file next to the sequence. Travels with the work.

Hardware-adaptive

Auto-tunes cache budget + worker count to your GPU / RAM. PBO upload on discrete GPUs, sync upload on integrated.

A timeline that tells you what's going on.

One bar carries every dimension of the sequence's state — at a glance, you know which frames are cached, which are missing, where you've left a note, what's in/out range, and where the playhead is.

  • Orange runs show frames already in RAM. The fill width is your prefetch buffer.
  • Red runs mark missing frames — Flick still plays through them with a checkerboard placeholder.
  • Green triangles = annotated frames. Blue dots = frames with comments.
  • Ctrl + click sets and drags the in / out points. Like Nuke.
  • The cursor is free to scrub outside the in/out range — playback snaps back inside on press.

Two ways to annotate.

Persistent mode = the marks you draw are saved to a sidecar JSON, frame-keyed, ready to share. Ghost mode (👻) = marks fade out on their own, perfect for live screen-share commentary in a Meet / Discord call.

  • 7-color review palette tuned for VFX (red / yellow / green / blue / orange / b&w).
  • Per-frame undo — Ctrl+Z only undoes the current frame's edits, no surprise jumps.
  • Eraser deletes whole strokes by hit-testing the polyline, not pixel-level.
  • Three fade presets in ghost mode: court ~2 s, moyen ~5 s (default), long ~10 s.
  • Bake into export if you want a self-contained MP4 to send a colleague.

Export to anything.

Send your sequence as an image-per-frame run for the next tool in the pipe, or bake it into one of eight video codecs to ship a review reel. Annotations toggle on / off; the OCIO display transform too.

  • Range: full sequence, in/out, or custom from / to.
  • Resolution: source / 720p / 1080p / 4K UHD / custom.
  • Frame rate: source / 23.976 / 24 / 25 / 29.97 / 30 / 60 / custom.
  • Color: bake the OCIO display transform, or write the linear buffer untouched (recommended for EXR + TIFF).
  • Annotations: bake into pixels + optional sidecar JSON copy alongside the export.

Real OCIO color, on the GPU.

Flick ships with the ACES 2.0 CG built-in config. Point it at your studio's custom config.ocio via the $OCIO env var if you have one. The display transform runs on the fragment shader, so toggling source / display / view is free.

  • Auto-detect input colorspace from filename + EXR header attributes.
  • Per-image exposure nudge with + / - (±0.25 stops).
  • Multichannel EXR: pick a single AOV / depth / normals layer and view it as RGB.
  • RGBA mute: toggle channels off in the shader without invalidating the cache.

Per-frame comments for the team.

Annotations are visual; comments are textual. A side panel beside the viewer holds a thread of notes attached to whatever frame the playhead is on — perfect for "the lighting is too cool" feedback that doesn't need a stroke.

  • Author auto-filled from the OS username — no login flow, comments just work.
  • Created / edited timestamps tracked per comment.
  • Blue dots on the timeline mark every commented frame so you can jump straight to the conversations.
  • Same JSON sidecar as annotations — one file travels with the sequence.
  • The Comments tab shares the side dock with the Color tab; flip with one click.
