v1.5.14 2026-05-18 Contact sheet mode + annotations re-architected per-layer + Save Frame As… full-source render.

New — Contact sheet mode

  • Tile every visible layer in a grid, all re-aligned to "frame 0" so you read across instead of around. Ctrl+G toggles; the master timeline still drives playback for the whole grid, with a per-tile drag-to-scrub override for shifting one layer's frame independently.
  • Smart auto-grid picks cols × rows that maximise per-tile area inside the current viewport aspect — no empty rails.
  • Axis presets ("N rows" / "N columns") replace the old fixed "1×2 / 2×3 / 4×4" presets. Pinning one axis auto-fits the other to the layer count, so 5 layers + "2 rows" becomes a 3×2 grid with one empty cell instead of silently dropping the 5th layer like a "1×2" fixed pick would.
  • Per-tile annotation bake — each tile composites its layer's strokes at its source frame via the export's paint_strokes helper. Existing notes appear across the grid at a glance.
  • Per-tile labels with #### placeholder substitution — render.1042.exr rather than render.####.exr · 1042.
  • Output divisor control (full / ½ / ¼ / ⅛ source res) trades GPU upload bandwidth for review-grade detail.
  • Transport integration: a dedicated grid-icon toggle button + a kebab "⋯" settings popup. Master timeline dims + locks while CS is active (per-tile scrub is the active gesture). Play buttons grey out — the per-layer scrub is the frame-control gesture in this mode.
  • Save Frame As… contact-sheet preset — when CS is active, the resolution combo auto-injects a Contact sheet (W×H) entry as the default, shipping the grid as a PNG / TIFF in one click.

New — Annotations & comments per-layer architecture

  • Strokes & comments now partition by layer.id. AnnotationStore and CommentStore keep a two-level layer_id → frame → state map internally; the public API stays frame-keyed via a current_layer_id attribute updated by the layer panel's focus signal. Existing call sites work unchanged.
  • Sidecar v2 (.img_player_annotations.json) keys every entry on layer.id with name_hint + source_path_hint for portability. The legacy v1 schema (keyed on basename under sequences) still loads — auto-migrated to v2 on first save. Multi-basename v1 files re-key the other basenames as legacy:<basename> so their data stays reachable.
  • Right-click on a layer → "Replace source…" opens a file picker, swaps the underlying sequence / video while preserving layer.id. Annotations + comments attached to that id stay; the user's trim is clamped to the new source's bounds and preserved verbatim when the numbering matches. Auto-detects video vs image-sequence by extension.
  • Cross-layer signals (layer_frame_annotated(str, int), layer_frame_comments_changed(str, int)) for consumers that need every layer's events (export bake, persistence dump).

New — Save Frame As… full-source render

  • Renders the full source image, not a viewport screenshot. Zoom + pan no longer crop the saved pixels — the whole frame is read at native resolution, OCIO-baked, optionally Lanczos-resized via OIIO.
  • Resolution picker: Source / 4K UHD / 1080p / 720p / Custom with Lock-aspect (same UX as the full Export dialog). Custom dims persist across sessions per user.
  • Contact-sheet preset injected at the top of the combo when CS mode is active (see above).

New — UX polish

  • Preferences dialog remembers the active tab across opens — within the session AND across launches (round-tripped via QSettings).
  • Timeline always shows a 100-frame skeleton when the session is empty, instead of collapsing to a blank rectangle. Short sequences keep their actual length — the widening only kicks in on truly degenerate ranges.
  • Contact-sheet mode always boots in normal display regardless of the last persisted enabled flag — opening a session that was closed in CS mode no longer lands the user on the grid view by surprise. The other CS state (grid choice, divisor, labels) is preserved.
  • Axis presets share the master x-axis with the layer bars — the 100-frame skeleton widening is applied at LayerPanel.broad_master_range (single source of truth) so the timeline and the bars never drift apart on short sequences.
  • Pause / resume the alt-channel background prefetch. A small status indicator sits right next to the channel selector — ▶ in green when the prefetch is running, ■ in red when it's stopped. Click flips the state: paused = no more disk I/O / decode / RAM churn for AOVs the reviewer isn't actively switching to; the active channel keeps decoding normally. Resume re-issues the full prefetch wave from the playhead.

Fixes

  • Still-as-topmost no longer bypasses multi-layer composite. The still anchor-alias fast path used to fire on every topmost.is_still case — including when the still had alpha_composite=True and a sequence covered the same frame below. Result: alpha holes showed the GL checker instead of the layer underneath. Now the fast path only fires when the still IS the sole contributor.
  • Replace-source cache flush is fully self-managed by MasterFrameCache. A new _last_known_source[layer.id] token (directory + base_name + padding + extension + frame count) is compared inside _on_layer_modified AND _on_layers_changed; mismatch drops the per-layer _path_index and clears the frame cache. Covers both the forward "Replace source…" path and the Ctrl+Z undo path (which goes through _restore() → layers_changed, not layer_modified).
  • Per-tile scrub mapping is letterbox / pillarbox aware. The old x / widget_w × cols formula assumed the composite filled the widget — true in smart-grid mode (matched aspects), false in manual grids where the composite is letterboxed inside the widget. Now reads the actual drawn rect from the same _zoom_factor / _pan_x/y the GL renderer paints from.
  • Per-tile scrub offset clamped to the layer's trim range so dragging past the end no longer accumulates an out-of-range value that has to be "unwound" before the tile starts moving back.
  • CS grid pushed to GL viewport from _render_contact_sheet (every render) rather than only from toggle_contact_sheet — boot-from-prefs case where CS is enabled at start now correctly routes mouse drags to per-tile scrub instead of falling back to the master-timeline scrub.
  • Contact-sheet button border grey at rest, accent only on hover / when checked — used to inherit the compare button's orange ACCENT_DIM border which made the cadre look permanently active.
  • Replace-source preserves the user's trim (clamped to the new source's bounds) and updates the layer name to the new basename — was overwriting the trim with the new full range and keeping the stale name.

Test suite grew from 971 to 1024 (+53 cases covering the new contact-sheet, annotation-persistence v2, layer-source-token, and resolution-picker code paths).

v1.5.13 2026-05-14 Smoother playback + the codebase grew up: +3.4% FPS, no more paint spikes, zero test debt.

Performance — hot-path tuning

  • OpenGL paint pipeline overhauled. The viewport used to re-resolve every shader uniform location (15+ per paint) and rebind every OCIO LUT to its texture unit on every frame. Now uniform locations are cached at program-link time, and the LUT samplers + bindings are pushed once when the OCIO bundle is applied. Result: paint mean −25 % (6 802 µs → 5 089 µs), paint p95 unchanged at 16 ms, and the worst-case paint spike collapsed from 593 ms to 47 ms (−92 %) — no more visible hoquets during playback.
  • OCIO shader bundle now LRU-cached on (manager, source, display, view, language). Toggling exposure / gamma used to silently rebuild a 384 kB 3D LUT every time even though those uniforms don't bake into the OCIO pipeline; now it's a dict hit.
  • Per-frame composite math allocates one buffer instead of three. The over-operator inner loop used to materialise arr * inv_a as a fresh ndarray per layer (~64 MiB at 4K); reused scratch buffer + in-place premultiply when the caller owns the read-frame output.
  • Cache-bar timer holds the lock once per tick, not once per cached key. cached_frames() / missing_frames() fire every 200 ms with O(N) lock acquisitions before; one acquisition now.
  • Scanner uses os.scandir() with cached DirEntry.stat() — ~3× faster than iterdir + Path.stat on Windows, more on networked / Drive-synced filesystems.
  • 17 lazy handler imports hoisted in app.py — the per-frame import dict-lookups were a measurable slice of the frame_changed handler at 24 fps.

Code health — long methods split, dead code removed, tests revived

  • 180-LOC _evict_if_over_budget split into a 25-LOC orchestrator + 5 named helpers (_eviction_distance_score, _signature_refs_invisible, _signature_is_beauty, _eviction_sort_key, _evict_in_order). The tiered "stale-sig → invisible-layer-only → live" policy is now scannable in one pass.
  • Frame-changed dispatch (app.py) split into _sync_per_frame_widgets / _try_compare_then_video / _try_display_cached_frame / _handle_no_coverage_gap / _handle_cache_miss — each path is its own named method instead of a 115-LOC if-cascade.
  • Stack-change refresh similarly split: _sync_compare_band_for_stack_change, _sync_navigable_range_to_layer_panel, _redisplay_current_frame_or_show_gap.
  • Shared cache constants + helpers live in cache/_common.py — CacheStats, DEFAULT_BUDGET_BYTES, BEHIND_PLAYHEAD_PENALTY, expected_filename are no longer duplicated between the single-layer and multi-layer cache flavours.
  • One canonical enrich_with_header in sequence/scanner replaces the two near-identical copies the live-load flow and the session loader were carrying.
  • One signature-token helper (_signature_token) replaces 3 hand-coded copies of the per-layer cache-key format.
  • Dead code removed: _source_key_at (70 LOC, never called since 1.5.1), VideoWriter.set_output_basename (broken-by-design API), various stranded headers + stale docstrings.
  • QSettings bool coercion deduplicated — preferences and export.settings were carrying separate copies of the same coercer with subtle drift. Both route through _value_coerce.qsettings_bool now.
  • Narrowed exception handling in audio, GL cleanup, PyAV decode paths — typed exceptions where the failure mode is known, defensive Exception with explicit log only at the outer boundary.
  • Full test suite now green: 971 passed / 0 failed / 0 errors. The 20 fails + 16 errors + 3 deselected tests that had been bit-rotting (Mock spec=FrameCache rejected new methods; tests using source-frame numbers as master-frame numbers; obsolete UI features) are now either revived or replaced with tests that match the documented contracts.

Methodology note: perf numbers measured on RTX 3080 / 32-thread CPU, 4K EXR sequence × 5 passes / 30 warmup frames, baseline vs Tier 1 warm-cache runs. Tier 2 + the test pass are zero-impact on perf (paint p50 / p95 identical) — the gains above all come from Tier 1. Cold-cache benches are also captured for traceability but the decode-worker background activity dominates the signal; see commit 1d3ae17 for the analysis.

v1.5.12 2026-05-14 Apply with no changes no longer materialises user TOML.

Fixes

  • Apply on Preferences > Color Management without editing any field used to write every value to the user flick.toml unconditionally — silently capturing the resolved (site / hardcoded) values as user-level overrides. The next site-config update would then be ignored on that machine. Setters are now properly gated behind explicit "did this field change?" checks. A regression test makes sure it stays that way.
v1.5.11 2026-05-13 One-shot QSettings → TOML migration, not per-launch.

Fixes

  • Legacy QSettings → user TOML migration was running every launch — re-importing the same registry values back into flick.toml even after the user had deleted it, making "fall back on site defaults" impossible. Now persists a _user_prefs_migration_version flag in QSettings after the first run; subsequent launches skip migration entirely, so deleting the user TOML really does reset to site / hardcoded defaults.

v1.5.5 2026-05-13 Disk-cache, hardened — robustness, multi-process safety, perf.

New

  • Shutdown drain budget 2s → 10s + visible "Flushing disk cache…" indicator so closing Flick with a fat writer backlog actually flushes everything instead of dropping the last few decoded frames.
  • Orphan blob sweep at boot — scan the cache tree for .bin files not referenced by SQLite and unlink them. Catches pre-1.5.1 bogus-key blobs + crash-mid-write leftovers + the user-deleted-index case.
  • Auto-reload via QFileSystemWatcher — debounced ~200 ms so a re-render burst coalesces into one reload. Picks up comp re-publishes without Ctrl+R.
  • Format-version migration via SQLite PRAGMA user_version — auto-wipe the cache once on format bump, no manual "Clear cache now".
  • Cross-process lock at <cache_dir>/.lock. A second Flick instance falls back to read-only mode (queries OK, no writes) with a banner in Preferences — prevents corruption from concurrent writers.
  • Perf v2 format — struct-packed 16-byte header replaces np.save's NPY format (~1.5× faster reads). Perf v3 format opt-in via a "Compress blobs (lz4)" toggle — raw bytes for fast NVMe, ~5× faster reads at 2× the disk space. Both formats coexist; the reader auto-routes via the blob's magic prefix.
v1.5.4 2026-05-12 Per-contributor disk caching.

New

  • Disk cache moves from composite-level to per-layer caching — each contributing layer's contribution (post-premult, pre-opaque-floor) is stored separately. Reorder / hide / add an unrelated layer → the other contributors stay hot on disk. The composite is rebuilt on the fly (~5-10 ms over-blend) instead of re-decoding every layer.
v1.5.3 2026-05-12 Pre-paint timeline for disk-available frames + live stats panel.

New

  • Pre-paint timeline — a dim-orange wash on the cache bar marks frames available on disk before you scrub to them. Bulk contains_keys SQLite query at session-load (~50 ms for a 1000-frame single-layer sequence).
  • Live disk-cache stats in Preferences > Disk cache — hits / misses / writes / read MB / written MB, refreshed every 1.5 s. Read-only banner appears when another instance owns the cache.
v1.5.2 2026-05-12 Disk-cache perf: native float16 reads + batched access updates.

New

  • Skip the float16 → float32 astype on read — the GL viewport accepts GL_HALF_FLOAT natively, so the cast was pure waste (~30 ms per 4K frame).
  • Batched last_access SQLite updates — every get() used to fire one UPDATE; we now coalesce 2 s of hits into one transaction. Eviction sees timestamps up to ~2 s stale (fine — eviction is rare).
v1.5.1 2026-05-12 Disk-cache wrong-channel bug fix.

Fixes

  • Disk cache source key was computed from live state at decode time, not from submit state — so a user toggling channels mid-decode could end up with the alt-channel pixels stored under the live-channel key. Mixed RGBA / albedo frames in some saved sessions. Fixed by capturing the key at submit time alongside the rest of the plan; format key bumped v1 → v2 to invalidate any corrupted blobs from v1.5.0.
v1.5.0 2026-05-12 Persistent disk-cache tier — close, reopen, scrub: cache hit.

New

  • 3-tier cache architecture — RAM → lz4-compressed half-float blobs on disk → source decode. Decoded frames evicted from RAM are persisted; opening the same shot tomorrow finds yesterday's frames warm.
  • SQLite WAL index for the disk tier (atomic, crash-safe). Sharded blob layout (<cache_dir>/ab/cd/<hash>.bin) avoids 100k files in a single directory.
  • Preferences > Disk cache — enable/disable, custom location (e.g. NVMe scratch), soft budget (default 50 GB, oldest entries evicted at 85% threshold), Clear button.
v1.4.5 2026-05-12 Hi-DPI splash polish.

Fixes

  • Splash now crisp on Hi-DPI displays — the bundled splash.png is now rendered at 2× the logical splash size (960×520 physical for a 480×260 logical splash) and the runtime loader tags the pixmap with devicePixelRatio(2.0). Qt draws it 1:1 on Hi-DPI monitors instead of upscaling, so the text and icon stay sharp. 1× monitors get a clean downscale instead of native-pixel jaggies.

v1.4.3 2026-05-11 Picker columns, layer right-click menu, truthful per-channel cache.

New

  • Column-laid sequence picker — multi-source picker now shows resolution, frame range, and a complete / partial / single-frame status badge per row in aligned columns (computed once from the longest text in each column so a 30-sequence drop reads cleanly). The "select only complete" button ticks multi-frame sequences with no missing frames in one click.

  • Beauty-pass-protected channel eviction — when RAM hits budget the cache now evicts entire channels FIFO (oldest cached channel first), keeping the beauty pass (RGB / RGBA) as the last to drop. Pre-cache now suspends itself when free budget < one frame's worth, instead of churning the worker pool.
  • Pure-offset drag is now a rekey for multi-layer stacks too — dragging a layer's offset slider used to invalidate every cached entry for that layer (full re-decode on release). Multi-layer stacks now reuse the single-layer rekey path: cached entries are remapped to the new master frames in place, no decode triggered.

  • Splash Loading… ghost — the bundled PNG no longer bakes a placeholder status string; only the runtime showMessage() paints, so the user no longer sees two overlapping "Loading Flick Player…" lines on cold boot.
  • Build chain — auto-regenerate splash PNG on each PyInstaller run (so a version bump can't drift behind the splash label) and drop a Flick Player v<version>.txt marker next to the .exe so a quick listing tells you which build you've got.
v1.4.2 2026-05-10 Video scrub fast-seek, channel-mode redesign, off-thread scan.

Fixes

  • Channel menu rows stay legible under the cache-fill bar at any fill percentage — text repaints in the cream #FFE5C0 tone clipped to the filled region, so the label reads cleanly across the seam between filled and empty parts of the bar.
v1.4.0 2026-05-09 Preferences dialog, OCIO hot-reload, splash refactor.

New

  • Preferences dialog — File → Open Preferences… (Ctrl+,) opens a sidebar+stack settings window. First section: Color Management, with three modes for resolving the OCIO config at boot — Default (built-in ACES), From $OCIO, or Custom config file… (browse to any .ocio on disk). The dialog's structure is set up to accept further sections (Playback, Cache, Annotation…) by adding one widget.
  • OCIO config hot-reload — applying a new config in Preferences swaps the running OCIOManager in place: ColorPanel combos repopulate, the GPU shader rebuilds against the new transform, persisted source / display / view picks that no longer exist in the new config are dropped silently. Restart no longer required. Banner turns green with the loaded config name and a summary of which picks were preserved vs reset.
  • Cream-on-orange channel menu typography — when the cache-fill bar is half-filled behind a channel row, the label text repaints in #FFE5C0 (the same warm cream the bottom info band uses) clipped to the filled region, so the row stays legible all the way across the bar. Outside the bar the standard theme colour shows through, so typography is untouched until loading begins.
  • QSplashScreen at boot — the previous PyInstaller bootloader splash used Tk, which isn't DPI-aware on Windows and visibly shrunk + repositioned the splash mid-boot on HiDPI displays. The new splash is a Qt QSplashScreen reading a checked-in PNG asset; DPI-aware, no shrink. Trade-off: ~1-2 s appearance latency (PySide6 import) instead of the bootloader's instant paint.

Fixes

  • Cache-fill bar on channel rows starts a few pixels before the label's first character (in the radio↔label gutter) so the orange wash never crashes flush against the leftmost glyph.
  • Sparse-sequence load freeze — dropping a sequence with many missing frames (typical of comp picks where only kept frames are rendered) used to freeze the UI for ~10 s on a 4 K range with ~50 holes. The cache was building a fresh 33 MB filename-baked placeholder per gap; switched to the shared memoised placeholder for sparse gaps (the per-frame variant is kept for runtime decode failures where the user actually benefits from seeing which path is bad). Drop time falls to ~2 s, matching the contiguous-sequence baseline.
  • Build chain — bundle build now produces a working EXE on miniforge Windows: tkinter excluded again (saves ~5 MB), Tcl/Tk PATH workaround dropped, OIIO / OpenEXR / Imath DLLs explicitly pulled from $CONDA_PREFIX\Library\bin\ so the bundle is no longer missing them at startup. build_exe.bat auto-zips the bundle to dist\FlickPlayer_v<version>.zip for portable distribution — atomic transfer, no install needed on the target machine.
v1.3.0 2026-05-08 A/B compare, smart cache, color profiles.

New

  • A/B compare overlay — pick two layers and split the viewer with a vertical wipe, horizontal wipe, opacity blend, or solo-B mode. The seam is draggable (right-click + drag), keyboard-nudgeable, and persists across layer picks. Compositing runs on the fragment shader so seam updates are uniform-only — no numpy compose, no cache invalidation.
  • Compare band in the menu bar — A / B picker pills, mode pills (Vert / Horiz / Opacity / Solo-B), and a swap button live next to the File menu. The band shrinks with the window width and never overlaps menu hit-targets.
  • Smart cache with multi-version key — the cache key is now (master_frame, signature) where the signature folds in layer id, label, active channel, premult flag, offset and in-frame. Switching channel or stack order keeps the previous version's entries hot for instant flip-back; eviction prefers older versions of off-screen frames first.
  • Background channel pre-cache — alt-channel reads queued at low priority during idle on every visible layer (top-down stack order), so swapping the active channel on any layer paints in cached frames immediately. Suspended automatically during playback to free the GIL, resumed on pause.
  • Per-row cache progress — the channel menu shows a dotted progress bar per channel and the layer button mirrors the active channel's coverage. The status bar tracks total RAM headroom.
  • Color metadata sweep — the loader matches EXR header attributes against Nuke / Arnold / V-Ray / Houdini patterns (substring sweep on colorspace, color_space, role / alias names) before falling back to filename heuristics.
  • Unmarked-EXR override — pick the default colorspace per pipeline in the Color panel (ACEScg, scene_linear, Linear Rec.709…) for EXRs that ship without metadata; persisted across sessions.
  • Solo-alpha + transparency BG picker — the alpha-only mute swaps in a checkerboard or a solid background colour so the matte reads correctly against the viewport.
  • Force reload — Ctrl+Shift+R clears the cache for the active master frame and re-reads from disk, even for frames already resident. Useful when artists overwrite shots in place.
  • Save Frame As… — File → Save Frame As… snapshots the viewer to PNG / JPG / TIFF / BMP / WebP with the same compare and annotation toggles as the full export.
  • Custom export filename + bake compare overlay — empty falls back to source basename, otherwise drives the output ({stem}.NNNN.ext for sequences, {stem}.ext for video). The live A/B wipe bakes through every frame; untick to ship the active sequence's clean composite.
  • Single-image layers — drop a still on the viewer and it becomes a layer with adjustable hold duration. Great for slates, refs and frame holds in a comp pass.
  • Versioned bundle directory — dist/FlickPlayer_v<version>/ instead of dist/img_player/, so multiple builds coexist.
  • No console window — the .exe ships without a stray cmd window. Logs are written to %LOCALAPPDATA%\img_player\flick.log with rotation.
  • Splash screen at startup — PyInstaller bootloader splash with three status milestones (Configuring logger…, Detecting hardware…, Loading Flick Player…) so cold launch on slow disks doesn't look frozen.
  • Taskbar icon — Windows AppUserModelID set explicitly so the taskbar shows flick.ico instead of the generic Python feather.
  • Channel arrow navigation — Up/Down arrows in the channel menu walk through the channel list without leaving the keyboard. Annotation close button + fullscreen polish ship in the same pass.
  • Missing-frame placeholder bakes the filename — the rich error graphic now writes the missing frame's expected filename into the overlay, so reviewers can spot which frame to re-render.
